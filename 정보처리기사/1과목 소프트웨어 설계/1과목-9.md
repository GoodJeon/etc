# 정보처리기사 공부(2022-01-23)



## 애플리케이션 설계 A

### 소프트웨어 아키텍처

* 소프트웨어의 기본 구조
* 구성 요소들 간의 관계
* 좋음 품질 유지, 비기능적/기능적 요구사항 구현



#### 소프트웨어 아키텍처의 설계

1. 모듈화
   * 나누기(모듈 단위로)
2. 추상화
   * 불필요한 부분을 생략하고 간략화 시키는 것
   * 과정 추상화 : 자세한 수행 과정에대해서 정의x, 전반적인 흐름만 파악할 수 있도록 설계
   * 데이터 추상화 : 데이터의 세부적인 속성이나용도는 정의x, 데이터의 구조를 대표하는 표현으로 대체
   * 제어 추상화 : 이벤트 발생 시키는 방법을 자세하게 정의x, 대표할 수 있는 표현으로 대체
3. 단계적 분해
   * 하향식 설계 방법
   * 예시 : 기능 -> 알고리즘, 자료구조 
4. 정보 은닉
   * 변경하지 못하도록 할 때
5.  소프트웨어 아키텍처의 품질 속성
   * 이해 관계자들이 요구하는 수준의 품질을 유지하고 보장할 수 있게 설계 되었는지 확인하기 위함
   * 시스템 측면 : 
     * 성능(빠르게 처리하는가?)
     * 보안(허용되지 않은 접근을 막아야함)
     *  가용성(장애 없이 정상적으로 서비스를 제공하느냐)
     * 기능성(사용자가 요구한 기능을 만족스럽게 구현하는가)
     * 사용성(헷갈리지 않고 편리하게 구현하는가)
   * 비즈니스 측면:
     * 시작 적시성(정해진 시간에 맞춰 프로그램을 출시)
     * 비용(적절한 비용을 쓰는지, 유지/보수를 위해 많은 비용이 들어가지 않는지)
     * 예상 시스템 수명(만약 다음 버전이 나온다면?)
   * 아키텍처 측면:
     * 개념적 무결성(현재 시스템과 이루는 구성요소들이 일관성을 유지하는가)
     * 정확성(사용자들의 요구사항을 모두 충족시켜 주는가)
     * 구축 가능성(모듈 단위로 구축된 시스템을 적절하게 분배해서 일정을 변경해서 구축할 수 있는가)
     * 변경, 적응성, 대체성 등
6. 소프트웨어 아키텍처의 설계과정
   1. 설계 목표 설정
      * 시스템의 개발 방향을 명확히 하기 위함
      * 우선 순위, 목표 설정
   2. 시스템 타입결정
      * 아키텍처의 패턴을 선택
   3. 아키텍처 패턴 적용
      * 시스템 표준
   4. 서브시스템 구체화
      * 상호작용
   5. 검토
      * 설계 목표에 부합하고 요구사항을 잘 반영되었나 검토







### 아키텍처 패턴

* 아키텍처를 설계할 때 참조할 수 있는 **전형적인 해결 방식**
* 기본적인 윤곽, 규칙, 서브시스템 간의 관계 포함
* 장점
  * 패턴이 있으면 만들기 쉬움
  * 개발시간 단축
  * 고품질의 소프트웨어 생산 가능
  * 안전적 개발 가능
  * 의사소통이 간편해짐
  * 구조 이해
* 종류
  * 레이어 패턴
    * 시스템을 레이어(Layer: 계층)로 구분
    * 가장 고전적 방법
    * OSI 참조 모델
  * 클라이언트-서버 패턴
    * 요청을 하고 요청에 응답해 정보를 제공하는 방식
    * 서버는 항상 대기상태여야 함
  * 파이프-필터 패턴
    * 각 단계를 필터 컴포넌트로 캡슐화
    * 파이프를 통해 데이터를 전송
  * 모델-뷰-컨트롤러 패턴
    * MVC(Model-View-Controller)
    * 모델 : 서브 시스템의 핵심 기능과 데이터를 보관하는 개념
    * 뷰 : 정보를 표시
    * 컨트롤러 : 사용자로부터 받는 입력을 처리하는 부분 
  * 기타
    * 마스터-슬레이브 패턴
    * 브로커 패턴(연결)
    * 피어-투-피어 패턴(자기자신이 클라이언트이면서 서버가 될 수 있는)
    * 이벤트-버스 패턴
    * 블랙보드 패턴(해결책: 음성인식, 챠랑 식별, 신호)
    * 인터프리터 패턴(프로그램 코드에 각 라인의 수행하는 방법을 지정)
